1. Elemente finden (Selektionen)

Bevor du ein Element verändern kannst, musst du es erst auswählen. Drei wichtige Methoden:

  • document.getElementById('id') → ein Element anhand seiner ID
  • document.querySelector('.klasse') → das erste Element das zum CSS-Selektor passt
  • document.querySelectorAll('button') → alle Elemente die passen (Liste)

2. Was ist ein Event?

Ein Event ist ein Signal: "Der Nutzer hat geklickt", "Eine Taste wurde gedrückt", "Die Maus ist über ein Element gefahren". Du kannst auf dieses Signal hören und dann Code ausführen.

Jeder Klick erhöht die Zahl. Das ist ein click-Event.

3. addEventListener – "Hör zu, wenn etwas passiert"

Die Methode element.addEventListener(event, funktion) sagt dem Browser: "Rufe diese Funktion auf, wenn das Event eintritt." 

meinButton.addEventListener('click', () => {
    alert('Geklickt!');
});

Hier erscheint deine Eingabe live.

Der Button hört auf click, das Eingabefeld auf input.

4. Das Event-Objekt

Jede Event-Funktion bekommt automatisch ein Event-Objekt übergeben. Es enthält Infos: Welches Element wurde geklickt? (event.target). Wurde die Seite neu geladen? Dann verhindere das mit event.preventDefault(). 

element.addEventListener('click', (event) => {
    console.log(event.target); // das geklickte Element
});

Klicke auf A, B, C oder D.

Ein einziger Event-Listener auf dem Eltern-Element – event.target sagt dir, welches Kind geklickt wurde. Das nennt man Event-Delegation.

6. Standard-Verhalten verhindern

Ein Klick auf einen Link lädt normalerweise eine neue Seite. Ein Absenden eines Formulars lädt die Seite neu. Mit event.preventDefault() verhinderst du das – genau wie bei eurem Quiz!

Das Formular wird nicht neu laden – event.preventDefault() verhindert das. Stattdessen wird der eingegebene Text unterhalb angezeigt.
